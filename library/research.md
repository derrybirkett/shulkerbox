# Research

Articles, papers, and writings on productivity systems, knowledge management, and tool design.

---

## Productivity Systems

### Getting Things Done (GTD)
**By David Allen**

Core principles we apply:
- Capture everything in a trusted system (inbox)
- Weekly review to process and plan
- Next actions are specific and actionable
- Contexts matter (what can you do now?)

How it influenced shulkerbox:
- inbox.md as capture mechanism
- weekly-review as review ritual
- friction.md as context-specific action items

---

### Building a Second Brain
**By Tiago Forte**

PARA method (Projects, Areas, Resources, Archives):
- Projects: Active work with deadlines
- Areas: Ongoing responsibilities
- Resources: Reference material
- Archives: Inactive items

How it influenced shulkerbox:
- notes/ as second brain structure
- skills-plan.md as project list
- library/ as resources
- reviews/ as archive of reflections

## Tool Design Philosophy

### The Unix Philosophy
**Rob Pike, Brian Kernighan**

Principles we follow:
1. Do one thing well
2. Work together via composition
3. Handle text streams (universal interface)
4. Silent on success

Applied to:
- Small, focused skills
- Skills that pipe together
- Markdown as universal format
- Hooks that stay silent

---

### The Pragmatic Programmer
**Andy Hunt, Dave Thomas**

Key concepts we use:
- DRY (Don't Repeat Yourself) → Skills capture repeated workflows
- Prototype to Learn → Build, test, prune
- Good Enough Software → Ship and iterate
- Stone Soup → Start small, grow organically

Applied to:
- Skill-first approach (codify patterns as skills)
- Weekly pruning ritual
- Minimal but complete templates
- v0.1.0 → evolve from there

## Knowledge Management

### Zettelkasten Method
**Niklas Luhmann**

Principles we borrow:
- Atomic notes (one idea per note)
- Links between notes
- Emerge structure over time
- Index for entry points

How it influenced shulkerbox:
- insights.md as atomic captures
- Cross-references in skills
- Organic growth over predefined taxonomy

**Various sources**

Core ideas:

Applied to:
- activity-log → weekly-review → skills-plan

## Systems Thinking

### Feedback Loops
**Donella Meadows - Thinking in Systems**

Concept: Systems improve through feedback loops

Applied to productivity-agent:
```
Work → Capture → Review → Learn → Improve → Better Work
  ↑                                              ↓
  └──────────────────────────────────────────────┘
```

Key insight: Without the review step, there's no improvement

---

### The Lean Startup
**Eric Ries**

Build-Measure-Learn cycle:
1. Build (minimal skill)
2. Measure (track usage in reviews)
3. Learn (what works, what doesn't)
4. Pivot or persevere

Applied to:
- Ship skills fast
- Prune unused skills (4-week cycle)
- Activity log as measurement
- Weekly review as learning
